## 0.12 cs、ds这类sreg段寄存器，位宽是多少

CPU内部的段寄存器（Segment reg）：

- CS：代码段寄存器（Code Segment Register），存放代码段的段基址。
- DS：数据段寄存器（Data Segment Register），存放数据段的段基址。
- ES：附加段寄存器（Extra Segment Register），存放附加数据段的段基址。称为附加，是因为此段寄存器用途不像其他sreg那样固定，可以额外做他用。
- FS：附加段寄存器。
- GS：附加段寄存器。
- SS：堆栈段寄存器（Stack Segment Register），存放堆栈段的段值。

## 0.28 MBR、EBR、DBR和OBR各是什么

计算机上电之后运行的是基本输入输出系统BIOS，BIOS只完成一些简单的检测或初始化工作，然后把处理器使用权交给MBR，MBR在固定的位置，位于整个硬盘最开始的扇区。

MBR是主引导记录，Master Boot Record（Main Boot Record），它存在于整个硬盘最开始的那个扇区，即0盘0道1扇区，这个扇区称为MBR引导扇区。

MBR引导扇区中的内容：

- 446字节的引导程序及参数
- 64字节的分区表：分区表中每个分区表项占16个字节，因此MBR分区表中可容纳4个分区。
- 2字节结束标记0x55和0xaa

活动分区标记位于分区表项中最开始的1字节，其值要么为0x80，要么为0。0x80表示此分区上由引导程序，0表示没有引导程序，该分区不可引导。MBR找到了活动分区，就将CPU使用权交给此分区上的引导程序，此引导程序通常是内核加载器。

内核加载器的入口地址在各分区最开始的扇区。该扇区称为操作系统引导扇区，其中的引导程序（内核加载器）称为操作系统引导记录（OS Boot Record，OBR），此扇区也称为OBR引导扇区。

DBR（DOS Boot Record），DOS操作系统的引导记录。

EBR（Expand Boot Record），为扩展分区存储分区表的扇区。

### 3.2.3 实模式下内存分段的由来

实模式的实体现在，程序中用到的地址都是真实的物理地址，段基址:段内偏移产生的逻辑地址就是物理地址，也就是程序员看到的完全是真实的内存。

### 3.2.6 实模式下的ret

CS寄存器是代码段的段基址，IP寄存器是代码段的段内偏移地址。

- ret（return）指令的功能是在栈顶（寄存器ss:sp所指向的地址）弹出2字节的内容来替换IP寄存器。
- ret只置换了IP寄存器，不用换段基址，属于近返回。
- ret指令也要负责维护栈顶指针。
- retf（return far）指令从栈顶渠道4字节，栈顶处的2字节用来替换IP寄存器，另外的2字节用来替换CS寄存器。
- retf属于远返回。
- retf也要负责维护栈顶指针。

### 3.2.7 实模式下的call

实模式下，call指令调用函数有4种方式：

- 16位实模式相对近调用
	- 近： call指令所调用目标函数和当前代码段是同一个段，只给出段内偏移地址就好，不用给出段基址。
	- 近调用用关键字near来修饰，near表示在内存或寄存器中取2字节。
	- 相对： 由于是在同一个代码段中，所以只要给出目标函数的相对地址即可。
- 16位实模式间接绝对近调用
	- 间接： 是指目标函数的地址没有直接给出，地址要么在寄存器中，要么在内存中，总之不以立即数的形式出现。
	- 绝对： 是指目标函数的地址是绝对地址。
	- 这也是近调用，即只能调用同一个代码段中的函数，只给出段内偏移就好，不用给出段基址。
- 16位实模式直接绝对远调用
	- 直接：就是操作数在指令中直接给出，是立即数。在各种转移指令中，凡是包含直接的，都不需要经过寄存器或内存，操作数以立即数的形式给出。
	- 远： 指要跨段，目标函数和当前指令不在同一个段中
- 16位实模式间接绝对远调用
	- 间接：段基址和段内偏移地址都不是立即数，要么在内存中，要么在寄存器中。
	- 间接绝对远调用的形式不支持寄存器寻址，只支持内存寻址，即段基址和段内偏移地址在内存中。
	- 内存中内容是4字节大小，内容是地址，前（低）2字节是段内偏移地址，后（高）2字节是段基址。
### 3.2.8 实模式下的jmp

5类转移方式：

- 16位实模式相对短转移
	- 相对：意味着操作数是个相对增量，有正负之分。
	- 相对短转移的机器码大小是2字节，操作码是1字节，故操作数是1字节，1字节有符号数表示范围-128~127, 很短。
	- 只在段内转移，不需跨段。
- 16位实模式相对近转移
	- 相对近转移和相对短转移相比，就是操作数范围增大了，由8位宽度变成了16位宽度。
- 16位实模式间接绝对近转移
	- 目标地址是绝对地址，并且未在指令中直接给出，存在寄存器或内存中。
- 16位实模式直接绝对远转移
	- 直接： 是指操作数不仅是立即数，而且CPU直接拿来就用，不用再转换。
	- 绝对： 是指提供的操作数是绝对地址。
	- 远： 是指目的地址和当前指令不是一个段，有跨段的需求，所以操作数要包括新的段基址和段内偏移。
- 16位实模式间接绝对远转移
	- 操作数不是直接给出，操作数只能在内存中。

### 3.2.9 标志寄存器flags

有条件转移的条件放在标志寄存器flags中。

以下标志位仅在8088以上CPU中有效：

- 第0位是CF位，即Carry Flag，进位。用于检测无符号数加减法是否有溢出，CF为1时，最高位有进位或借位。
- 1、3、5、15位没有专门的标志位，空着占位用。
- 第2位为PF位，即Parity Flag，奇偶位。用于标记结果低8位中的1的个数，如果为偶数，PG位为1，否则为0。
- 第4位为AF位，即Auxiliary carry Flag，辅助进位标志，用来记录运算结果低4位的进、借位情况，若低半字节有进、借位，AF为1，否则为0。
- 第6位为ZF位，即Zero Flag，零标志位。若计算结果为0，此标志为1，否则为0。
- 第7位为SF位，即Sign Flag，符号标志位。若运算结果为负，则SF位为1，否则为0。
- 第8位为TF位，即Trap Flag，陷阱标志位。此位若为1，用于让CPU进入单步运行方式，若为0，则为连续工作的方式。平时用的debug程序，在单步调试时，原理上就是让TF位为1。
- 第9位为IF位，即Interrupt Flag，中断标志位。若IF位为1，表示中断开启，CPU可以响应外部可屏蔽中断。若为0,表示中断关闭，CPU不再响应来自CPU外部的可屏蔽中断，但CPU内部的异常还是要响应的。
- 第10位为DF位，即Direction Flag，方向标志位。此标志位用于字符串操作指令中，当DF为1时，指令中的操作数地址会自动减少一个单位，当DF为0时，指令中的操作数地址会自动增加一个单位，意即给地址的变化提供个方向。单位的大小取决于用什么指令。
- 第11位为OF位，即Overflow Flag，溢出标志位。用来标识计算的结果是否超过了数据类型可表示的范围。若OF为1，表示有溢出，为0表示未发生溢出。专门用于检测有符号数运算结果是否有溢出现象。

以下标志位仅在80286以上CPU中有效，相对于8088，它支持特权级和多任务：

- 第12-13位为IOPL，即Input Output Privilege Level，用在有特权级概念的CPU中。有4个任务特权级，即特权级0、特权级1、特权级2、特权级3。 IOPL要用2位来表示这4种特权级。
- 第14位为NT，即Nest Task，任务嵌套标志位。8088支持多任务，一个任务就是一个进程。当一个任务中又嵌套调用了另一个任务时，此NT位为1，否则为0。

以下标志位仅用于80386以上的CPU：

- 第16位为RF位，即Resume Flag，恢复标志位。该标志位用于程序调试，指示是否接受调试故障，它需要与调试寄存器一起使用。RF为1时忽略调试故障，为0时接受。
- 第17位为VM位，即Virtual 8086 Model，虚拟8086模式。这是实模式向保护模式过度时的产物，现在已经没有了。

以下标志位仅用于80486以上的CPU：

- 第18位为AC位，即Alignment Check，对齐检查。对齐是指程序中的数据或指令器内存地址是否是偶数，是否是16、32的整数倍，没有余数，这样硬件每次对地址以自增的方式访问内存时，自增后的地址正好对齐数据所在的起始地址上，这就是对齐的原理。若AC位为1，则进行地址对齐检查，为0时不检查。

以下标志位只对80586（奔腾）以上CPU有效：

- 第19位为VIF位，Virtual Interrupt Flag，虚拟中断标志位。虚拟模式下的中断标志。
- 第20位为VIP位，Virtual Interrupt Pending，虚拟中断挂起标志位。在多任务情况下，为操作系统提供的虚拟中断挂起信息，需要与VIF位配合。
- 第21位为ID位，Identification，识别标志位。系统经常要判断CPU型号，若ID为1，表示当前CPU支持CPU id指令，这样能获取CPU的型号、厂商等信息。若ID为0，表示当前CPU不支持CPU id指令。
- 剩下的22-31位都没有实际用途，占位用。

### 3.3.1 CPU如何与外设通信-IO接口

IO接口的功能：

- 设置数据缓冲，解决CPU与外设的速度不匹配。
- 设置信号电平转换电路。CPU用的信号是TTL电平，外设大多是机电设备，不能使用TTL电平驱动，可在接口电路中设置电平转换电路来解决。
- 设置数据格式转换。CPU只能处理数字信号。数字信号需要经过数/模转换（D/A）成模拟量才能被送到外设以驱动硬件，模拟量也需要经过模/数（A/D）转换成数字量才能被CPU处理。
- 设置时序控制电路来同步CPU和外部设备。
- 提供地址译码。


- 输入输出控制中心（I/O Control hub，ICH），南桥芯片。用于连接pci、pci-express、AGP等低速设备。
- 北桥芯片。用于连接高速设备，如内存。

指令：

- in：用于从端口中读取数据
- out：用于往端口中写数据
### 3.3.3 显存、显卡、显示器

显卡（显示适配器）有自己的BIOS，位置是0xC0000到0xC7FFF。显卡支持三种模式：

- 文本模式
- 黑白图形模式
- 彩色图形模式

从0xB8000到0xBFFFF这片32KB大小的内存区域是用于文本显式。

每个字符在屏幕上都是由连续的2个字节来表示的，每个字符的低字节是字符的ASCII码，高字节是字符属性元信息。

在高字节中低4位是字符前景色，高4位是字符的背景色。颜色用RGB红绿蓝三种基色调和，第4位用来控制亮度，若置1则呈高亮，若位0则为一般正常亮度值。第7位用来控制字符是否闪烁。

## 3.4 bochs调试方法

- x，用来查看内存，x命令后接线性地址。
- xp，用来查看内存，xp命令后接物理地址。
- xp指令： `xp/nuf <addr>`
	- n：指定要显示的显示单元数
	- u：指显示单元的大小
	- f：指要使用哪种进制显式。
- bochs中的字是4字节，用b、h、w、g来选择一个显示单元的大小：
	- b：1字节
	- h：半个字，2字节
	- w：一个字，4字节
	- g：双字，8字节
- u命令：将内存数据反汇编成指令。

- Debugger control类指令：
	- `q、quit、exit`，退出调试状态，关闭虚拟机
	- `set`
		- 设置寄存器的值：`set reg = val`，可设置通用寄存器和段寄存器。
		- 设置每次停止执行时，是否反汇编指令：`set u on|off`。
	- `show`
		- `show mode`：每次CPU变换模式时提示。
		- `show int`：每次有中断时提示。同时显式三种中断类型，包括`softint、extint、iret`。
		- `show softint`：只显示软件主动触发的中断。
		- `show extint`：只显示来自外部设备的中断。
		- `show iret`：只显示iretd指令有关的信息。
		- `show call`：每次由函数调用发生时提示。
	- `trace on|off`：如果设置为on，每执行一条指令，都会将反汇编的代码打印到控制台。
	- `u | disasm [/num] [start] [end]`：将物理地址start到end之间的代码反汇编。如果不指定地址，则反汇编EIP指向的内存。num指定反汇编的指令数。
	- `set size = 16|32|64`在使用反汇编命令时，用来告诉调试器段的大小。
	- `ctrl + c`：中断执行，回到bochs控制台。
- Execution control类：
	- `c、cont、continue`：向下继续执行。
	- `s|step [count]`L：执行count条指令，count是指定单步执行的指令数，若不指定，count默认为1。
	- `p|n|next`：执行1条指令，若待执行的指令时函数调用，则把整个函数当作一个整体来执行。
- Break point management类：
	- `vb|vbreak [seg: off]`：以虚拟地址添加断点，程序执行到此虚拟地址时停下来。
	- `lb|lbreak [addr]`：以线性地址添加断点，程序执行到此线性地址时停下来。

## 3.5 硬盘介绍

### 3.5.3 硬盘控制器端口

端口就是位于IO控制器上的寄存器。

硬盘控制器主要端口寄存器：

|                         | Primary通道 IO端口 | Secondary通道 IO端口 | 读操作时端口用途         | 写操作时端口用途       |
| :---------------------- | :------------- | :--------------- | :--------------- | -------------- |
| Command Block registers | 0x1F0          | 0x170            | Data             | Data           |
|                         | 0x1F1          | 0x171            | Error            | Features       |
|                         | 0x1F2          | 0x172            | Sector count     | Sector count   |
|                         | 0x1F3          | 0x173            | LBA low          | LBA low        |
|                         | 0x1F4          | 0x174            | LBA mid          | LBA mid        |
|                         | 0x1F5          | 0x175            | LBA high         | LBA high       |
|                         | 0x1F6          | 0x176            | Device           | device         |
|                         | 0x1F7          | 0x177            | Status           | Command        |
|                         |                |                  |                  |                |
| Control Block registers | 0x3F6          | 0x376            | Alternate status | Device Control |

端口分为两组：
- Command Block registers：用于向硬盘驱动器写入命令字或从硬盘控制器获得硬盘状态。
- Control Block registers：用于控制硬盘工作状态。

端口是按通道给出的。

硬盘中的扇区在物理上是用“柱面-磁头-扇区”来定位的（Cylinder Header Sector），简称为CHS。

一种逻辑上位扇区编址的方法，称为LBA（Logical Block Address），逻辑块地址。有两种：

- LBA28，用28位比特来描述一个扇区的地址。最大寻址范围是 $2^{28}$个扇区，每个扇区是512字节，最大支持128GB。
- LBA48，用48位比特来描述一个扇区的地址。最大寻址范围是 $2^48$个扇区，每个扇区是512字节，最大支持128PB。

LBA寄存器：

- LBA low：8位宽度，用来存储28地址的0-7位。
- LBA mid：8位宽度，用来存储28位地址的8-15位。
- LBA high：8位宽度，用来存储28位地址的16-23位。

LBA寄存器总共能存储24位地址，其余4位地址存放在device寄存器，device寄存器宽度是8位，此寄存器的低4位用来存储LBA地址的第24-27位。

device寄存器：
- 0-3位：用来存储LBA地址的第24-27位。
- 第4位：用来指定通道上的主盘或从盘，0代表主盘，1代表从盘。
- 第5位：固定为1，称为MBS位。
- 第6位：用来设置是否启用LBA方式，1代表启用LBA模式，0代表启用CHS模式。
- 第7位：固定为1，称为MBS位。

### 3.5.4 常用的硬盘操作方法

常用的数据传送方式：

- 无条件传送方式：数据源设备一定时随时准备好了数据，CPU随时取都没问题，如寄存器、内存就是类似这样的设备。
- 查询传送方式：也称为程序I/O、PIO（Programming Input/Output Model），是指传输之前，由程序先去检测设备状态。数据源设备在一定的条件下才能传送数据，这类设备通常是低速设备，比CPU慢很多。对硬盘可用此方式来获取数据。
	- 缺点：CPU需要不断查询设备状态。
- 中断传送方式：也称中断驱动I/O。当数据源设备准备好数据后，通过发送中断来通知CPU拿数据。
- 直接存储器存取方式（DMA）：不让CPU参与传输，完全由数据源设备和内存直接传输，CPU直接到内存中拿数据就好。DMA由硬件实现，需要DMA控制器。
- I/O处理机传送方式：CPU可以不知道有传输这回事。需要单独硬件来支持。

# 第4章 保护模式入门

## 4.1 保护模式概述

### 4.1.1 为什么要有保护模式

实模式：

- 实模式下操作系统和用户程序属于同一特权级。
- 用户程序引用的地址都是指向真实的物理地址，即逻辑地址等于物理地址。
- 用户程序可以自由修改段基址，也可以访问所有内存。
- 访问超过64KB的内存区域要切换段基址。
- 一次只能运行一个程序，无法充分利用计算机资源。
- 共20条地址线，最大可用内存为1MB。

### 4.1.2 实模式不是32位CPU，变成了16位

32位CPU具有保护模式和实模式两种运行模式，可以兼容实模式下的程序。

## 4.2 初见保护模式

### 4.2.1 保护模式之寄存器扩展

寄存器要保持向下兼容，必须在原有的基础上扩展（extend），各寄存器在原有16位基础上，再次向高位扩展了16位，称为了32位寄存器。经过extend后的寄存器，统一在名字前加e表示扩展。

6个段寄存器，依然是16位。

全局描述符表，其中每个表项称为段描述符，大小位64字节，用来描述各个内存段的起始地址、大小、权限等信息。全局描述符表很大，放在内存中，由GDTR寄存器指向它。

段寄存器中保存的不再是段基址，而是selector（选择子），该selector其实是个数，用这个数来索引全局描述符表中的段描述符。

为了提高获取段信息的效率，将段信息用寄存器来缓存，这就是段描述符缓冲寄存器（Descriptor Cache Registers）。

### 4.2.2 保护模式之寻址扩展

实模式下内存寻址中，基址寻址、变址寻址、基址变址寻址这三种形式中的基址寄存器只能是bx、bp，变址寄存器只能是si、di。其中bx默认的段寄存器是ds，经常用于访问数据段，bp默认的段寄存器是ss，经常用于访问栈。

实模式下的寄存器有固定的使命。对于寻址中的偏移量，只能是1个字以内的立即数，即不能超过16位。

保护模式下内存寻址中，基址寄存器是所有32通用寄存器，变址寄存器是除了esp之外的所有32位通用寄存器，偏移量变成了32位，还可以对变址寄存器乘以一个比例因子，比例因子只能是1、2、4、8。

### 4.2.3 保护模式之运行模式反转

指令格式：`| 前缀 | 操作码 | 寻址方式、操作数类型 | 立即数 | 偏移量 |`。

编译器提供伪指令`bits`，格式：

- `[bits 16]`: 告诉编译器，下面的代码编译成16位的机器码。
- `[bits 32]`: 告诉编译器，下面的代码编译成32位的机器码。

`bits`指令的范围是从当前bits标签直到下一个bits标签的范围。bits外面的中括号可以省略。在未使用bits指令的地方，默认是`[bits 16]`。

指令格式中，前缀字段里面存放的是指令选项之类的东西，比如：

- 指令重复前缀rep
- 段跨越前缀“段寄存器:”
- 操作数反转前缀0x66
- 寻址方式反转前缀0x67

如果要用另一种模式下的操作数大小，需要在指令前添加指令前缀0x66，将当前模式临时改变成另一种模式。

在指令中添加了0x66反转前缀之后：

- 如果当前运行模式是16位实模式，操作数大小将变成32位。
- 如果当前运行模式是32位保护模式，操作数大小将变成16位。
转换只是临时的，只在当前指令有效。

## 4.3 全局描述符表

全局描述符表（Global Descriptor Table，GDT）是保护模式下内存段的登记表。

### 4.3.1 段描述符

段描述符，用来描述一个内存段，该结构是8字节大小：
- 高32位：
	- 31~24:  段基址31~24
	- 23: G，Granularity，粒度，用来指定段界限的单位大小。若G为0，表示段界限的单位是1字节，若G为1，表示段界限的单位是4KB。
	- 22: D/B，用来只是有效地址（段内偏移地址）及操作数的大小。
	- 21: L，用来设置是否是64位代码段。L为1表示64为代码段，否则表示32位代码段。
	- 20: AVL，Available
	- 19~16: 段界限19~16
	- 15: P，Present，段是否存在。如果段存在于内存中，P为1，否则P为0。
	- 14~13: DPL，Descriptor Privilege Level，描述符特权级，能表示4种特权级：
		- 0: 最高特权级，操作系统
		- 1
		- 2
		- 3: 最小特权级，用户程序。
	- 12: S
		- 0: 系统段
		- 1: 数据段
	- 11~8: TYPE，共4位，用来指定本描述符的类型
	- 7~0: 段基址23~16
- 低32位：
	- 31~16: 段基址15~0
	- 15~0: 段界限15~0

### 4.3.2 全局描述符表GDT、局部描述符表LDT及选择子
- 全局描述符表GDT（Global Descriptor Table）。
- GDTR（GDT Register）寄存器，用来存储GDT的内存地址及大小。GDTR是48位寄存器。
	- 0~15位是GDT界限值，以字节为单位。
	- 15~47是GDT内存起始地址。
	- 操作GDTR寄存器要使用lgdt指令。
- 段的选择子selector，在保护模式下，段寄存器中存放的是选择子。16位。
	- 0~1位存储RPL，即请求特权级，可表示0、1、2、3四种特权级。
	- 2位是TI位，即Table Indicator，用来指示选择子是在GDT中，还是LDT中索引描述符。
		- 0表示在GDT中索引描述符。
		- 1表示在LDT中索引描述符。
	- 3~15位是描述符的索引值，用此值在GDT中索引描述符。13位，可索引 $2^{13} = 8192$个段。
- 局部描述符表LDT（Local Descriptor Table），CPU厂商为在硬件一级原生支持多任务而创造的表，一个任务对应一个LDT。
- LDTR（LDT Register）寄存器，用lldt指令操作。

### 5.4.1 特权级那点事

特权级：

- 0：操作系统。
- 1：系统程序，一般是虚拟机、驱动程序等系统服务。
- 2：系统程序，一般是虚拟机、驱动程序等系统服务。
- 3：用户程序。

### 5.4.2 TSS简介

TSS，Task State Segment，任务状态段，是处理器在硬件上原生支持多任务的一种实现方式。TSS是一种数据结构，用于存储任务的环境。

特权级转移分两类：

- 由中断门、调用门等手段实现低特权级转向高特权级。
- 由调用返回指令从高特权级返回到低特权级。

TSS由TR（Task Register）寄存器加载。

### 5.4.3 CPL和DPL入门

保护模式下有段描述符，访问内存得先经过它，它的作用是通过各种属性描述一段内存区域。

在保护模式下，段寄存器中存储的是选择子，选择子的第0~1位是RPL字段。RPL：请求特权级。

指令请求、访问其他资源的能力等级称为请求特权级，指令存放在代码段中，所以用代码段寄存器CS中选择子的RPL位表示代码请求别人资源能力的等级。

代码段寄存器CS和指令指针寄存器EIP中指向的指令是当前在处理器中正在运行的代码，所以，位于CS寄存器中选择子低2位的值不仅称为请求特权级，又称为处理器的当权特权级。

当前特权级CPL（Current Privilege Level）。

当前正在运行的代码所在的代码段的特权级DPL就是处理器的当前特权级。

段描述符中有一个属性为该内存标明了特权等级，DPL字段，Descriptor Privilege Level，描述符特权级。

对于受访者位数据段来说，只有访问者的权限大于等于该DPL表示的最低权限才能继续访问。

对于受访者位代码段来说，只有访问者权限等于该DPL表示的最低权限才能够继续访问，即只能平级访问。

### 5.4.4 门、调用门与RPL序

处理器只有通过门结构才能由低特权级转移到高特权级。门结构就是记录一段程序起始地址的描述符。

门描述符同段描述符类似，是8字节大小的数据结构，用来描述门中通向的代码。由4种门结构：

- 任务门描述符，可放在GDT、LDT和IDT中。
- 中断门描述符，仅位于IDT中。
- 陷阱门描述符，仅位于IDT中。
- 调用门描述符，可放在GDT、LDT中。

### 7.3.1 外部中断

外部中断，又称硬件中断。

CPU提供两条信号线：

- INTR（INTeRrupt）。外部设备。可屏蔽中断。
- NMI（Nom Maskable Interrupt）。灾难性错误。不可屏蔽中断。

### 7.3.2 内部中断

内部中断可分为：
- 软中断，软件主动发起的中断。
	- `int 8位立即数`：通过该中断进行系统调用。
	- `int3`：调试断点指令，中断向量号是3。
	- `into`：中断溢出指令，中断向量号是4。
	- `bound`：检查数组索引越界指令，中断向量号是5。
	- `ud2`：未定义指令，中断向量号是6。
- 异常
	- Fault：故障，可被修复，是最轻的一种异常。缺页异常Page Fault。
	- Trap：陷阱，通常用在调试中。
	- Abort：终止，最严重的异常类型，无法修复。

## 7.4 中断描述符表

中断描述符表（Interrupt Descriptor Table，IDT），是保护模式下用于存储中断处理程序入口的表。

实模式下存储中断处理程序入口的表叫中断向量表（Interrupt Vector Table，IVT）。

# 第13章 编写硬盘驱动程序

## 13.1 硬盘及分区表

### 13.1.1 创建从盘及获取安装的磁盘数

物理地址0x475处存储着主机上安装的硬盘的数量，由BIOS检测并写入。

### 13.1.2 创建磁盘分区表

磁盘：
- 盘片
- 扇区
- 磁道
- 磁头
- 柱面
- 分区：多个编号连续的柱面组成。

硬盘MBR中有个64字节固定大小的数据结构，就是分区表，分区表中每个表项就是一个分区的描述符，表项大小是16字节，因此64字节的分区表总共可容纳4个表项，这就是为什么硬盘仅支持4个分区的原因。

分区表必须存在于MBR引导扇区或EBR引导扇区中。在这512字节中，前446字节是硬盘的参数和引导程序，然后是64字节的分区表，最后2字节是魔数55aa。

### 13.1.3 磁盘分区表浅析

磁盘分区表（Disk Partition Table），简称DPT，是由多个分区元信息汇成的表，表中每一个表项都对应一个分区，主要记录各分区的起始扇区地址，大小界限等。

最初磁盘分区表位于MBR引导扇区中。

MBR（Main Boot Record）主引导记录，它是一段引导程序，其所在扇区称为主引导扇区，该扇区位于0盘0道1扇区，也就是硬盘最开始的扇区，扇区大小为512字节，这512字节由三部分组成：

- 主引导记录MBR。共446字节，其中包括硬盘参数及部分指令（由BIOS跳入执行），它是由分区工具产生，独立于任何操作系统。
- 磁盘分区表DPT。共64字节大小，每个分区表项是16字节，因此磁盘分区表最大支持4个分区。
- 魔数55aa。共2个字节。
